# QSP Stage212  
**CI-Linked Evidence Bundle Pipeline**

MIT License © 2025 Motohiro Suzuki

---

## Overview

Stage212 introduces an **automated Evidence Bundle pipeline** linked directly to CI runs.

Each CI execution produces a reproducible package containing verification logs, CI metadata, and integrity manifests.  
The bundle is uploaded as a GitHub Actions artifact, allowing reviewers to download and inspect the verification evidence associated with a specific CI run.

This stage moves QSP from isolated test outputs to **CI-bound reproducible evidence packages**.

---

## Evidence Structure

The verification structure follows a traceable chain:


Claim
↓
CI Job
↓
Evidence Artifact
↓
CI Run ID
↓
Evidence Bundle


Each CI execution generates an evidence package tied to the run identifier.

---

## Evidence Bundle Contents

The generated bundle contains the following files:


actions_runs.json
actions_jobs.json
replay_attack.log
downgrade_attack.log
session_integrity.log
fail_closed.log
sha256sums.txt
summary.md


### actions_runs.json
CI run metadata including repository and run ID.

### actions_jobs.json
Results of security-related CI jobs.

### attack logs
Execution logs for simulated attack scenarios:

- replay attack
- downgrade attack
- session integrity validation
- fail-closed behavior

### sha256sums.txt
Integrity manifest for all files contained in the bundle.

### summary.md
Human-readable summary of the CI run and verification results.

---

## Automated Generation

Evidence bundles are automatically generated using:


tools/generate_evidence_bundle.py


This script collects CI outputs and logs from:


out/ci/
out/logs/


and builds a reproducible verification package.

---

## CI Integration

The pipeline is executed through GitHub Actions.

Workflow:


.github/workflows/stage212-ci.yml


During each CI run the system:

1. Creates CI metadata
2. Generates attack test logs
3. Builds the evidence bundle
4. Calculates SHA256 integrity manifest
5. Uploads the bundle as a GitHub Actions artifact

Artifacts can be downloaded from the **Actions** tab.

---

## Example Artifact

Artifacts generated by CI runs:


evidence-bundle-<run_id>.zip


Example:


evidence-bundle-22942171729.zip


The artifact contains the complete evidence bundle for that CI execution.

---

## Purpose

The Evidence Bundle provides:

- reproducible verification artifacts  
- traceability between claims and CI results  
- a portable package for external security review  

This structure supports security audits and independent validation.
